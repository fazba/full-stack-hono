## 任务目标

在 **Cloudflare Workers** 环境中，使用 **Hono** 框架实现一套完整的后端身份认证系统。

---

## 技术栈要求

| 组件                      | 用途                       |
| ------------------------- | -------------------------- |
| Cloudflare Workers + Hono | 运行时与 Web 框架          |
| D1 SQL Database           | 持久化存储用户数据（核心） |
| Workers KV                | 存储 Session / Token       |
| Secrets Store             | 存储密钥与敏感配置         |

---

## 功能需求

### 1. 用户名密码认证

- 用户注册（邮箱/用户名 + 密码）
- 密码登录
- 密码需加密存储（如 bcrypt / argon2）

### 2. Google OAuth 登录

- 支持 Google 账户授权登录
- OAuth 2.0 标准流程实现
- 自动关联/创建本地用户记录

### 3. 会话管理

- 登录后签发 Token / Session
- Token 存入 Workers KV，设置合理过期时间
- 支持登出（销毁 Token）

---

## 数据库设计要求

请合理设计 D1 中的数据表结构，至少包含：

- `users` 表（用户核心信息）
- `oauth_accounts` 表（第三方账号关联）

---

## 安全要求

- 密钥统一通过 **Secrets Store** 注入，禁止硬编码
- 实现基础防护（如重复注册校验、登录失败处理等）
